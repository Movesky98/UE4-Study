30. 고급 언리얼 모션 그래픽(UMG) 위젯 레이아웃

# UI에서 캔버스(Canvas) 이외의 레이아웃을 배치하는 다양한 방법을 살펴보자.
- 캔버스에 버튼을 나란히 배치하는 방법과 더불어
- 유연한 레이아웃을 만들 수 있다.
- UI 크기가 유동적인 변화에 반응할 수 있음

# 레이아웃 작동 방식
- 컨테이너 위젯이라는 개념과 함께 동작하고 있다.

# 컨테이너 위젯의 작동 방식
- Panel을 보면 Canvas, Grid 등 많은 Panel이 있는 것을 알 수 있다.
- Canvas Panel은 위젯 블루프린트를 생성하면 함께 생성되어 있는 모습을 볼 수 있음.
- Canvas와 Button 같은 Element의 차이점은, Button과 같은 Element는 하나의 자식만 가질 수 있다는 점이다.

- 이처럼 일부는 많은 슬롯(자식)을 가질 수 있고 버튼과 같이 제한된 경우도 있음
- 위젯 블루프린트의 Element가 가지고 있는 디테일 창에서 볼 수 있듯이,
  '슬롯(부모 위젯이름)'이 있는 것을 볼 수 있다. 
  (또, 어떤 부모를 두냐에 따라 디테일창이 다르다)
- 이 슬롯은 부모위젯에 의해 생성되기 때문에 그렇다고 함 ㅇㅇ

# Vertical Box / Horizontal Box / Size Box
- Box를 이용해 Canvas Panel 내의 레이아웃에서 좀 더 다양한 활동을 할 수 있다.
- Size Box를 이용해 위젯 사이의 여백으로 사용할 수 있다.

# Overlay
- 위젯 간의 깊이 값 (어떤 위젯을 더 앞으로 나오게 할 것인가) 를 조정할 때 유용하다.
- 자식들의 순서로 깊이값을 정할 수 있나봄 (아닐수도 찾아봐야 함)
