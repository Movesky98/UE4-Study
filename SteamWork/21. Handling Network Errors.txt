# 목표
- 게임 인스턴스를 여러개 생성하여 같은 세션에 입장한 후, 서버가 게임을 종료할 경우
  클라이언트들은 멈춰버리는 오류를 처리해보자.
- 서버 연결해제에 대한 네트워크 오류를 처리해보기

# 클라이언트가 연결을 끊는 네트워크 오류의 처리
- 세션 인터페이스를 사용하지 않고 엔진 객체에 대해 직접 이벤트를 구독하자
- 왜 엔진 객체를 이용하느냐! 실제로 모든 연결 로직은 엔진 수준에서 발생하기 때문
- 실제로 세션 관리자에서 발생하지 않고 그저 연결과 Connection String을 얻는 것이 전부
- 일단 다른 서버에 연결되면 대부분의 작업을 처리하는 것은 엔진이며,
  이는 네트워크 오류를 위한 다른 위치를 가지는 이유임.
 
 Q. 어떻게 엔진 객체에 접근할 것인가?
